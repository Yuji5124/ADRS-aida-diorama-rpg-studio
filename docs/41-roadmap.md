# 41 - ロードマップ (Roadmap)

> ADRS Demo 01 を **20〜30 回のやり取り** で完成させるための進行計画。
> 1 ターン = 1 回のやり取り（仕様確定 or 実装単位）。**実装は仕様合意の後** に行う。

---

## 全体像（5 フェーズ）

| フェーズ | 内容 | ターン目安 |
|---|---|---|
| 0. 実装前（仕様確定） | スコープ・データ・イベント・台本・技術方針 | 5〜6 |
| 1. 基礎実装 | マップ表示・移動・カメラ | 5〜6 |
| 2. RPG 機能（最大の山） | イベントエンジン・会話・宝箱・ワープ・戦闘 | 8〜10 |
| 3. AI サポート | NPC セリフ提案（1 機能） | 2〜3 |
| 4. デモ完成 | 導線結合・テストプレイ・演出最小強化 | 3〜4 |
| 合計 | | **23〜29** |

> **成功の分岐点はフェーズ 2 のイベントエンジン。**
> 「汎用コマンド列」として綺麗に作れれば、会話/宝箱/ワープ/戦闘が芋づる式に完成する。
> 個別実装すると 30 ターンでは終わらない。

## リスクと対策

| リスク | 影響 | 対策 |
|---|---|---|
| ジオラマ描画に時間を吸われる | RPG が遊べない箱になる | 表現はライト1灯+影で固定。霧/DOF は禁止 |
| イベントを個別実装してしまう | 工数爆発 | commands 統一を厳守（`21` 参照） |
| ジョブ×スキルの組み合わせ爆発 | バランス調整で溶ける | ジョブ2 / スキル各2 / 敵2 を超えない |
| DB 編集 GUI に着手 | Demo 1 本分の工数 | Demo 01 は JSON 手書き / 固定データ |
| AI を多機能化 | 制御不能・遅延 | 1 機能・承認制を厳守 |
