# 00 - ADRS 概要 (Overview)

> このドキュメントは ADRS / AIDA Diorama RPG Studio の全体像を定義する起点ドキュメントです。
> 本フェーズは **仕様検討フェーズ** であり、実装コードは含みません。

---

## ADRS とは

**ADRS（読み: エイドラス / AIDA Diorama RPG Studio）** は、
ドット絵タイルと 3D ジオラマ表現を組み合わせて、**短編 RPG を制作できるアプリケーション** です。

- **タグライン: 「ドット絵の世界を、光るジオラマRPGに。」**

RPGツクールMV 級の制作体験を目指しつつ、ADRS では 2D マップを 3D ジオラマとして立ち上げ、
**光・影・霧・奥行き** によって「作った瞬間に見栄えする RPG 制作ツール」を目指します。

---

## 名称ポリシー（重要）

ADRS では **「HD-2D」という名称は使いません。** 代わりに以下の表現を使います。

| 使う表現 | 用途 |
|---|---|
| ピクセルジオラマRPG | 日本語・一般向け |
| 2.5DジオラマRPG | 技術的な文脈 |
| Pixel Diorama RPG | 英語・一般向け |
| Diorama RPG Studio | プロダクト名としての説明 |

> ❌ 「HD-2D」表記は仕様書・UI・コメントいずれにおいても使用しないこと。

---

## コンセプトの3本柱

1. **ドット絵タイル** — 2D のピクセルアートで世界を描く
2. **ジオラマ立ち上げ** — 2D マップを 3D の立体ジオラマとして表示する（光・影・奥行き）
3. **作った瞬間に見栄えする** — 凝った設定なしで、配置しただけで絵になる

---

## 用語（最小グロッサリ）

| 用語 | 意味 |
|---|---|
| ジオラマ | 2D タイルマップを立体化して見せる ADRS の表示形態 |
| タイル | マップを構成するドット絵の最小単位 |
| イベント | マップ上に置かれた「動作するもの」。会話・宝箱・ワープ・戦闘開始などを `commands` で表現する |
| コマンド | イベントが実行する 1 ステップの命令（例: `message`, `transfer`） |
| アクター | プレイヤーが操作する／パーティに加わるキャラクター |
| ジョブ | アクターの職業。能力やスキルを規定する |
| トループ | 戦闘に登場する敵の編成 |
| フラグ | ゲーム進行状態を記録する真偽値（例: 宝箱の開封済み） |

> 用語の拡張版は将来 `docs/02-glossary.md` に分離する。
