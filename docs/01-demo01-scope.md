# 01 - Demo 01 スコープ (Scope)

> ADRS Demo 01 で「何を作り、何を作らないか」を確定するドキュメント。
> 迷ったら本書に立ち返る。**機能網羅ではなく、一周遊べる縦切りを優先する。**

---

## 成功条件（Demo 01 のゴール）

Demo 01 は「RPGツクールMV 級の機能網羅」ではなく、
**「短い RPG がタイトルからクリアまで一周遊べる縦切り（vertical slice）」** を成功条件とする。

具体的なプレイ導線:

1. タイトル画面から開始する
2. はじまりの村マップに入る
3. 主人公を移動できる
4. NPC と会話できる
5. 宝箱を開けてアイテムを入手できる
6. 洞窟へワープできる
7. 敵シンボルに触れると戦闘になる
8. ターン制戦闘で敵を倒せる
9. ボスを倒すとクリア画面になる

> この 1〜9 が最初から最後まで途切れず通ることが Demo 01 の合格ライン。

---

## Demo 01 の規模（確定）

| 項目 | 規模 |
|---|---|
| マップ | 2 枚（はじまりの村 / 小さな洞窟） |
| プレイ時間 | 3〜5 分 |
| 主人公 | 1 人 |
| 仲間システム | なし |
| ジョブ | 2 種類まで（ドット剣士 / 光術士） |
| スキル | 各ジョブ 2 個まで |
| 敵 | 通常敵 1 種 + ボス 1 種 |
| AI サポート | 1 機能のみ（NPC セリフ・クエスト文の提案） |

---

## ✅ Demo 01 でやること（MUST）

### マップ・移動
- ジオラママップ表示（2D タイル → 3D 立ち上げ）
- カメラは **固定の斜め見下ろし**
- グリッドベースのマップ
- 主人公のグリッド移動 + 衝突判定
- カメラの主人公追従
- 最小タイルセット: 草 / 道 / 水 / 壁 / 木 / 家 / 洞窟床 / 岩

### ジオラマ表現（最小）
- ライト 1 灯
- 簡易な影

### イベント（統一コマンド方式）
- 会話 / 宝箱 / ワープ / 戦闘開始 を **すべて Event の `commands` で表現**
- 最小コマンド: `message` / `addItem` / `setFlag` / `transfer` / `startBattle` / `clearGame`
- **フラグ（flag）** 3 個: 宝箱開封 `chest_potion_opened` / 通常敵撃破 `enemy_slime_01_defeated` / クリア記録 `boss_defeated`（未定義 flag は false）

### 戦闘
- ターン制コマンドバトル
- コマンド: `attack` / `skill` / `escape`
- 味方 1 人 vs 敵 1〜2 体
- 行動順は「味方 → 敵」固定
- ダメージ式は固定
- HP / MP / 勝利 / 敗北 / 逃走 を扱う

### ジョブ・スキル
- ジョブ 2 種（ドット剣士 / 光術士）
- 各 2 スキル
- **New Game 後にジョブを 1 つ選ぶ**（開始時ジョブ選択）。選択後は固定。**ゲーム中のジョブチェンジはなし**（→ `23-job-skill-spec.md`）

### AI サポート
- NPC セリフ・クエスト文の提案（1 機能のみ）
- 用途を選ぶ → セリフ案を 3 つ提示 → 採用したものだけ反映
- **自動反映しない**（必ず人間が承認）
- 本物の AI API 接続は後回しでよい（擬似 AI・テンプレート生成で可）

### 全体
- タイトル画面（New Game → マップ遷移）
- クリア画面
- テストプレイ（エディタからそのまま再生）

## ❌ Demo 01 でやらないこと（WONT）

| カテゴリ | やらないこと | いつやるか |
|---|---|---|
| カメラ | 回転カメラ | 後回し |
| ジオラマ | 霧 / 被写界深度 / 天候 / 複数光源 | 演出強化フェーズ |
| パーティ | 仲間システム / 隊列 | 後続デモ |
| ジョブ | ジョブチェンジ / 3 種以上 | 後続デモ |
| 戦闘 | 状態異常 / 属性 / 複雑な敵 AI / 速度順行動 | 後続デモ |
| 装備 | 武器 / 防具 / 装備変更 | 後続デモ |
| 経済 | ショップ / 売買 | 後続デモ |
| 永続化 | セーブ / ロード | 後続デモ |
| 成長 | 複雑なレベルアップ曲線 | 後続デモ |
| イベント | 条件分岐 / 変数 / マルチページ複雑分岐 | 後続デモ（フラグのみ Demo 01 で導入） |
| DB | DB 編集 GUI | 後続デモ（Demo 01 は JSON 手書き / 固定データ） |
| AI | 2 機能目以降 / 自動反映 / 実 API 必須化 | 後続デモ |

## スコープ判断の原則

1. **縦切り優先** — 「全機能が半分」より「最小機能が最後まで通る」
2. **イベントは統一** — 会話・宝箱・ワープ・戦闘を個別実装にしない
3. **数を絞る** — ジョブ 2 / スキル各 2 / 敵 2 を超えない
4. **見栄えは最小で達成** — ライト 1 灯 + 影で十分とみなす
5. **AI は 1 機能・承認制** — 暴走させない

> 新しい機能要望が出たら、まず「Demo 01 でやらないこと」に入れられないかを検討する。
